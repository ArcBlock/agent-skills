# Live Document

> 状态：产品思路阶段，AI Vibe Coding 时代正在出现的新出版形态

## 一句话定位

**Live Document 是一种"表现得像应用，但本质是文档"的出版物**

目的不是完成任务，而是：
- 表达一个判断
- 呈现一组洞察
- 让读者在作者设定的范围内探索

---

## 为什么我们一直需要但做不到

过去的"替代方案"：

| 方案 | 优点 | 问题 |
|------|------|------|
| **BI Dashboard + 截图** | 快速 | 一旦截图，所有东西都死了；看的人无法理解你是怎么得出结论的 |
| **Excel** | 可计算、可交互、可探索 | 容易被误操作破坏；原始数据和结论混在一起；很难只把"洞察层"分享 |
| **传统文章/Blog** | 可以讲清楚观点 | 图表是静态的；读者只能"相信你"，无法自己探索 |

> Live Document 解决的正是这些方案共同的缺陷

---

## "Live" 的真正含义

**不是**指数据实时刷新。

恰恰相反，大多数 Live Document 的关键特征是：
- 数据是 **snapshot**
- 有明确的**时间语义**（例如：2024 年 8 月分析）
- 不追求向后兼容
- 不追求"永远最新"

**为什么这反而是优势**：
- 洞察是阶段性的
- 判断是基于"当时我们知道的东西"
- 历史不应该被未来的方法重写

> Live Document 保存的是"当时的理解"，而不是"一个永远正确的系统"

---

## 与 App / Dashboard 的本质区别

| 对比项 | App / Dashboard | Live Document |
|--------|-----------------|---------------|
| **目标** | 长期运行 | 表达洞察 |
| **数据** | 动态 / 可重算 | Snapshot |
| **兼容性** | 非常重要 | 不重要 |
| **用户心智** | 使用 | 阅读 + 探索 |
| **生命周期** | 需要维护 | 发布即完成 |

> 真正想做"完整应用"的人不应该选择 Live Document；而 Marketing / 内容 / PM 会天然被它吸引

---

## 为什么 AI 让 Live Document 第一次成为可能

在没有 AI / Vibe Coding 之前，做交互式内容需要：
- 前端工程
- 数据工程
- UI 设计

**对个人来说几乎不可能**。

现在 AI 可以：
- 自动生成 UI
- 自动选择图表
- 自动构建交互
- 自动帮你"把理解变成结构"

> "表达洞察"的成本第一次降到了个人级别

---

## 典型使用场景

如果你做过这些，你已经在使用 Live Document：
- 把多个平台的数据整理成一个可交互页面
- 用 Web Coding 做过分析报告
- 把读书笔记、方法论、总结做成"一页网站"
- 做过可以点来点去的市场分析页

---

## 为什么需要"出版"而不是"部署"

常见担心："这些 Web Coding 的东西放在 SaaS 平台上，哪天平台没了怎么办？"

这和当年从 BBS 走向 Blog 的过程一模一样：
- 创作开始累积
- 人开始珍惜自己的作品
- 开始在乎长期存在、可控、可整理

> 一旦创作者在乎"这是我的作品"，就会天然地需要一个"出版空间"，而不是 Playground

---

## Live Document 是新的个人资产形态

**不是**：
- 临时 Demo
- 玩具应用
- 随手实验

**而是**：
- 思考轨迹
- 方法论沉淀
- 能力证明
- 可以被分享、被引用、被积累的作品

---

## 媒介升级的历史规律

| 时代 | 升级 |
|------|------|
| Blog | 让个人可以发布**思想** |
| Video / Podcast | 让个人可以发布**声音和影像** |
| **Live Document** | 让个人可以发布**可执行的理解** |

> 这不是工具升级，而是表达载体的升级

---

## 与业务策略的关系

Live Document 与 Vibe 应用部署的业务增长策略非常 match：
- 用户创作的 Live Document 需要出版空间
- 这正是 Blocklet 可以提供的价值
- 从 Playground → 出版 = 用户粘性提升

## 核心结论

> **Live Document 不是"更炫的网页"，而是 AI 时代，洞察与理解的原生出版形式。**
>
> 它让原本只存在于某个人脑子里的判断，第一次可以被结构化、交互化、长期保存下来。

## Agent 摘要

```
Live Document: A new publishing format in the AI Vibe Coding era.
"Looks like an app, but is essentially a document."

Purpose: Express insights, not complete tasks.
Data is snapshot with clear time semantics, not real-time.
Lifecycle: Publish and done, no maintenance needed.

Why now possible: AI reduces cost of interactive content to individual level.
Why "publish" not "deploy": Creators want ownership, not playground.

Media evolution:
- Blog = publish thoughts
- Video/Podcast = publish voice/image
- Live Document = publish executable understanding

This is not tool upgrade, but expression medium upgrade.
Matches Vibe app deployment growth strategy.
```

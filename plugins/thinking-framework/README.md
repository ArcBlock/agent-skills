# Thinking Framework Plugin

ArcBlock's technical thinking framework and proposal review methodology.

## Skills

### what-robert-thinks

A comprehensive skill for evaluating proposals, technical decisions, and product directions against ArcBlock's architectural principles and engineering philosophy.

**Use when:**
- Reviewing technical proposals before submission
- Evaluating new technologies or approaches
- Checking alignment with AFS/AINE architecture
- Self-reviewing designs for completeness and rigor

**What it includes:**
- Core thinking patterns and time horizons
- AFS (Agentic File System) principles
- AINE (AI Native Engineering) framework
- Blocklet/Chamber/Scaffold architecture
- DID/Capability model
- Product positioning (ArcSphere, Agent Fleet)
- Key technical judgments
- Self-review checklists
- Anti-patterns to avoid

## Installation

```bash
/plugin install thinking-framework@arcblock-agent-skills
```

## Usage

Simply mention "what robert thinks" or reference the skill when you need to evaluate a proposal or technical decision.

Example:
```
Using the what-robert-thinks framework, evaluate this proposal: [your proposal]
```

## Contributing

This skill reflects ArcBlock's evolving technical philosophy. Updates should be discussed with the team before merging.
